**⚠️ Include only one `Toaster` component per application** to ensure that only one toast is rendered at a time.

### New toasts replace existing toasts

`Toaster` only shows one toast at a time. Anytime `content` or `type` is added or modified, the current Toast will slide
down, and the new Toast with the updated content / type will slide up. Any other modified props will change the Toast in
place, without triggering re-animation.

**There's a lot you can do with Toast. Refer to the props of Toast for the full details.**
