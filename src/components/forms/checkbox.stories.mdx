# Checkbox

Renders an FDS Icon in place of the user agent checkbox. The user agent `input
type="checkbox"` is visually hidden behind the icon.

The `name` prop is required, as it is used to create a reference between the `label` and
`input` elements.

<hr />

<Story name="Knobs">
  <Checkbox
    indeterminate={boolean('indeterminate', false)}
    disabled={boolean('disabled', false)}
    name="default-checked"
    label={text('label', 'Checkbox label')}
  />
</Story>

## Managing checked state

This component uses the `onChange` event of the underlying `input type="checkbox"` to
manage checked state. To check the box by default, pass the `checked` boolean prop.

**Users can still uncheck the box.** To "freeze" checked state, use a disabled checkbox
(see below).

<Preview>
  <Story name="Managing checked state">
    <Checkbox checked name="default-checked" label="Checked by default" />
  </Story>
</Preview>

## Preventing user interaction

Use the `disabled` prop to prevent the user from changing the chekced state of the
checkbox.

<Preview>
  <Story name="Disabled checkboxes">
    <React.Fragment>
      <div>
        <Checkbox disabled name="disabled-unchecked" label="Disabled unchecked" />
      </div>
      <div>
        <Checkbox disabled checked name="disabled-checked" label="Disabled checked" />
      </div>
    </React.Fragment>
  </Story>
</Preview>

## Indeterminate checkbox

Passing the `indeterminate` boolean prop will set `indeterminate` attribute of the
underlying input element and render the `CheckIndeterminate` icon when the checkbox is
checked.

  </Story>
</Preview>
